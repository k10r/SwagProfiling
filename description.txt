<h1>Entwickler Toolbar</h1>
<p>Die Entwicklertoolbar erleichtert das Leben jedes Shopware Entwicklers. Sie liefert im Frontend Informationen, die Entwickler sonst immer selbst in den tiefen von Shopware zusammen suchen mussten.
So werden als Beispiel alle verfügbaren Smarty Variabeln oder alle aufgetreten Events chronologisch dargestellt.</p>
<br>
<h2>Komprimierte Ansicht</h2>
<p>In der komprimierten Ansicht der Toolbar, werden die ermittelten Daten summiert und komprimiert dargestellt. So ist auf einem Blick ersichtlich wie viele Datenbankabfragen stattgefunden haben, wie viele Events gefeuert wurden oder wie viele Templates geladen wurden.</p>
<br>
<h2>Detailierte Ansicht</h2>
<p>Die detailierte Ansicht der Toolbar ist über einen klick auf die Toolbarleiste am unteren Rand des Shops erreichbar. Hier werden die einzelnen Bereiche detailiert dargestellt.</p>
<br>
<h3>Config</h3>
<p>Der Config Bereich der Toolbar liefert genaue Einblicke in die geladene Shopware Konfiguration. So lassen sich zur Laufzeit ganz einfach die Shop Konfigurationen überprüfen.</p>
<br>
<h3>Request</h3>
<p>Der Request Bereich liefert alle Daten zu dem aktuellen Request und dessen Response.</p>
<br>
<h3>Session</h3>
<p>Hier sind alle Daten zu sehen, die Shopware in der aktuellen Session vorbehält</p>
<br>
<h3>Template</h3>
<p>Der Template Bereich ist vor allem für Entwickler interessant, die im Frontend Anpassungen durchführen wollen. Hier werden zum einen die registrierten Template Verzeichnisse angezeigt und zusätzlich alle im Template verfügbaren Daten dargestellt.</p>
<br>
<h3>Events</h3>
<p>Shopware Erweiterungen werden hauptsächlich über Events eingebunden. Doch welche Events verwendet werden können ist nicht jedem direkt ersichtlich. Im Events Bereich der Toolbar ist genau dargestellt welche Events im System aufgetreten sind und welche Daten in das Event rein gereicht wurden und wie diese möglicherweise durch andere Plugins bereits modifiziert wurden.</p>
<br>
<h3>Queries</h3>
<p>Der Query Bereich listet, wie der Name schon sagt, alle Datenbankabfragen (Queries) auf, die im aktuellen Request auf der Datenbank ausgeführt wurden.</p>
<br>
<h3>Emails</h3>
<p>Sollte in Shopware eine Email über das Shopware_Components_Mail Objekt versendet worden sein, so werden die Mails hier untereinander aufgelistet</p>
<br>
<h3>Cache</h3>
<p>Das Thema Cache ist für manche ein Misterium. Im Cache Bereich der Toolbar ist genau ersichtlich was, seit wann und wie lange noch sich etwas im Cache befindet.</p>
<br>
<h3>Exception</h3>
<p>Sollte in Shopware eine Exception aufgetreten sein, welche an das Response Objekt des aktuellen Request weiter geben wurde, wird diese Exception hier aufgelistet.</p>
<br>
<h3>PHP</h3>
<p>Der PHP Bereich der Toolbar stellt die Informationen von phpinfo() da.</p>
<br>
<h3>Trace</h3>
<p>Im Trace Bereich sind alle Funktionen des Shopware Stacks chronologisch dargestellt.</p>
<br>
<h3>Ajax Request</h3>
<p>Dieser Bereich listet alle Ajax Requests auf, die auf der aktuellen Seite aufgerufen wurden.</p>
